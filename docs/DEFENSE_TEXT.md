# Текст доклада для защиты

Здравствуйте, уважаемые члены комиссии.

Тема моего дипломного проекта — **разработка интеллектуальной системы мониторинга, прогнозирования и рекомендации парковочных мест на примере города Бишкек**.

---

## 1. Актуальность

Проблема парковок актуальна для крупных и развивающихся городов. Водители часто тратят время на поиск свободного места, особенно рядом с торговыми центрами, университетами, школами, офисами и жилыми районами.

Обычные парковочные системы могут показывать текущее количество свободных мест, но они не всегда позволяют заранее понять, насколько парковка будет загружена через определённое время.

Поэтому в моём проекте реализована система, которая не только отображает парковки, но и прогнозирует их загруженность, а также рекомендует пользователю подходящее место.

---

## 2. Цель проекта

Цель проекта — создать веб-систему, которая помогает пользователю найти парковочное место и позволяет администратору управлять парковками, зонами, пользователями, инфраструктурой и ML-моделью.

Основные задачи проекта:

1. Разработать backend API.
2. Реализовать frontend интерфейс.
3. Создать базу данных PostgreSQL.
4. Добавить роли администратора и водителя.
5. Реализовать карту парковок.
6. Реализовать аналитику.
7. Реализовать прогнозирование загруженности.
8. Реализовать рекомендации парковок.
9. Добавить учет инфраструктуры рядом с парковкой.

---

## 3. Использованные технологии

Backend реализован на FastAPI.  
В качестве базы данных используется PostgreSQL.  
Для работы с базой используется SQLAlchemy в асинхронном стиле.  
Авторизация реализована через JWT.

Frontend реализован на React и Vite.  
Для карты используется Leaflet и OpenStreetMap.  
Для графиков используется Chart.js.

Для машинного обучения используется библиотека scikit-learn и модель LinearRegression.

---

## 4. Роли пользователей

В системе есть две роли:

**Администратор** и **водитель**.

Администратор может управлять парковками, зонами, пользователями, запросами, объектами рядом с парковкой, а также запускать генерацию истории и обучение модели.

Водитель может просматривать парковки, смотреть карту, отправлять запрос “Ищу место” и получать рекомендацию.

---

## 5. База данных

В базе данных есть таблицы для пользователей, ролей, парковок, зон, истории загруженности, запросов, прогнозов, рекомендаций, уведомлений и метрик модели.

Основная таблица — `parkings`.  
В ней хранится информация о парковке: название, адрес, координаты, количество мест, занятые места, свободные места и тип зоны.

Для учета инфраструктуры добавлена таблица `nearby_objects`.

В ней хранятся объекты рядом с парковкой:

- университеты;
- школы;
- торговые центры;
- кафе;
- кинотеатры;
- офисы;
- рынки;
- жилые объекты.

Каждый объект имеет тип, расстояние до парковки, время работы, дни активности и вес влияния.

---

## 6. Учет инфраструктуры

Главная особенность проекта — учет инфраструктуры рядом с парковкой.

Например:

- если парковка находится рядом с вузом, то в будний день утром и днем загрузка будет выше;
- если парковка рядом с торговым центром, то вечером и в выходные спрос будет выше;
- если парковка находится в офисной зоне, то она сильнее загружена в рабочее время;
- если парковка находится в жилом районе, то вечером и ночью загрузка выше.

В системе это реализовано через:

```text
zone_type
day_of_week
hour
nearby_objects
active_objects_weight
requests_last_hour
